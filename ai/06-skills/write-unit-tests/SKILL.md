# Objetivo
Criar testes unitários úteis, pequenos e proporcionais ao risco.

# Quando usar
- Após criação de ViewModel
- Após criação de reducers/helpers
- Ao estabilizar fluxos de estado

# Regras
- Cobrir estado inicial
- Cobrir intents principais
- Evitar testes frágeis
